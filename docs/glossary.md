# Glossary

- **Adapter**: Chain-specific implementation that maps AOXCON protocol behavior to runtime-native primitives.
- **ActorId**: Canonical identity reference of the message originator.
- **Canonical**: Representation and behavior that is normative and chain-independent.
- **Conformance**: Degree to which an implementation matches all normative AOXCON requirements.
- **Domain Separator**: Context binding value used to prevent cross-domain signature/proof replay.
- **Envelope**: Canonical message metadata object used for cross-chain processing.
- **Idempotency**: Property that repeated processing of same `message_id` does not produce new side effects.
- **ProofBundle**: Serialized witness/proof material used for verification.
- **State Transition**: Deterministic movement from one protocol state to another.
